# 昔涟语音优化路线

## 核心问题

同一段文字，命令行生成 TTS 效果好于聊天生成。原因：

```
聊天：DeepSeek 写长句书面语 → TTS 模型（训于游戏口语短句）不会念 → 效果差
命令行：人手写短句 → 匹配训练数据分布 → 效果好
```

**根因不是 TTS 出 bug，是 DeepSeek 写的句子与训练数据句式分布不匹配。**

## 解决方案

两路并行：

**短期：文本归一化**——在 TTS 前把 DeepSeek 的长句拆成训练数据风格的口语短句。改 `cleanText` 加拆句逻辑，读分钟内生效。

**长期：补数据**——收集 TTS 表现差的句式，从 3.5h 原始素材找对应片段补训。模型没见过的东西永远不会。

---

## 当前状态

- 训练数据：2211 条（来自 3.5h 原始素材）
- 模型：GPT-SoVITS v2，SoVITS 82MB + GPT 155MB
- 推理：CPU，~3s/句，完全离线
- 效果：大部分自然，偶尔某些句式/词汇不够像

## 优化路线

### 一、文本归一化（立即，0 成本）

在 `server/services/localTTS.js` 的 `cleanText` 中加强：

```js
// 拆长句书面语 → 口语短句
text = text.replace(/但是|可是|不过/g, '。')
           .replace(/因为|所以|因此/g, '')
           .replace(/虽然|尽管/g, '')
```

目的：让 DeepSeek 偶尔写的书面长句先拆成训练数据风格的短句再喂 TTS。

### 二、补盲区数据（周期，2~4h/次）

**流程：**
1. 聊天中记录 TTS 效果不好的句子（句式/词汇）
2. 回原始 3.5h 素材找对应片段 → 校对 → 加入训练集
3. 被筛掉的 ~150 条优先捡回来
4. 重新训练 SoVITS + GPT，覆盖原有权重

**触发条件：** 攒够 30~50 条问题句

### 三、合成数据扩充（辅助，1h/次）

**用途：** 用 DeepSeek + 命令行 TTS 批量生产候选数据，补特定盲区。

```
DeepSeek 生成 200 句目标句式
       ↓
命令行逐条生成 WAV
       ↓
人工筛选 50 条最像的
       ↓
文本+音频加入训练集
       ↓
重训
```

**注意：** 合成数据只补盲区，2211 条真实数据是基础。不能只用合成数据训。

### 四、真人录音（长期，持续）

1. 模仿昔涟声线录 50~100 条高发句式
2. 混入训练集重训
3. 声音像不像取决于模仿水平
4. 攒够 200~300 条可考虑替换部分原始数据

---

## 优先级

| 优先级 | 方法 | 投入 | 效果 |
|:---:|------|:---:|:---:|
| P0 | 文本归一化 | 1h 改代码 | 立竿见影 |
| P1 | 捡回筛掉数据 | 2~4h | 明显 |
| P2 | 合成数据补盲区 | 1h/次 | 辅助 |
| P3 | 真人录音 | 长期 | 根本性提升 |
